# ERP 简易 Demo 设计文档

## 1. 项目概述

- **项目名称**: ai_erp 进销存 Demo
- **目标用户**: 企业内部管理人员（仓库管理员 / 采购员 / 销售员）
- **技术栈**: React (Vite) + JSON 文件（Mock 数据）
- **状态管理**: Zustand
- **核心流程**: 采购 → 销售 → 库存（进销存）
- **安全说明**: Demo 仅供演示，不含认证/权限系统

## 2. 用户故事

| 角色 | 需求 | 价值 |
|-----|------|-----|
| 仓库管理员 | 想在销售出库时被拦截，避免库存出现负数 | 防止超卖，保护库存准确性 |
| 采购员 | 希望能记录采购单价，以便对比不同供应商的成本 | 优化采购决策 |
| 销售员 | 想快速查看哪些商品库存不足需要补货 | 及时补货，避免断货 |
| 财务 | 想在 Dashboard 看到今日销售额 | 快速了解当日业务情况 |
| 仓库管理员 | 想查看每笔库存变动的来源订单 | 便于追溯库存异常 |

## 3. 核心模块

### 3.1 商品管理
- 商品列表（增删改查）
- 商品字段：名称、单价、库存、预警库存

### 3.2 采购管理
- 供应商列表（增删改查）
- 采购订单创建（含单价）
- 采购入库（更新库存）

### 3.3 销售管理
- 客户列表（增删改查）
- 销售订单创建（含单价）
- 销售出库（更新库存）
- **库存不足处理**: 提示"库存不足"，阻止出库

### 3.4 库存管理
- 库存查询（所有商品当前库存）
- 库存预警（低于阈值标红提示）
- 库存流水（进出记录，含时间戳）

## 6. 核心流程

### 采购入库流程
1. 选择供应商 → 添加商品和数量（含采购单价）→ 创建订单 → status: pending
2. 点击"入库" → 商品库存增加 → status: completed
3. 生成 inbound 类型库存流水（orderType: "purchase"）

### 销售出库流程
1. 选择客户 → 添加商品和数量（含销售单价）→ 创建订单 → status: pending
2. 点击"出库" → **检查库存是否充足**
   - 库存不足: 弹窗提示"库存不足，当前库存 X 件"
   - 库存充足: 扣减库存 → status: completed → 生成 outbound 类型库存流水（orderType: "sale"）

### 库存预警流程
- 在 /inventory 页面，库存 < minStock 的商品标红显示

## 7. 状态与约束

- 库存不能为负数（出库前必须检查）
- 出库时库存不足则阻止操作
- 订单 status: completed 后不可修改
- 使用 Zustand 管理前端状态

## 9. MVP 边界（v1 只做这些）

**包含**:
- 商品/供应商/客户的增删改查
- 采购订单创建 + 入库
- 销售订单创建 + 出库（含库存检查）
- Dashboard 统计
- 库存预警

**不包含** (后续版本):
- 订单编辑/取消
- 退货/退款
- 多用户权限
- 报表导出
